Product Service Master - Travel Module

Comprehensive guide for configuring travel services for B2B and D2C models

Overview

The Product Service Master is a core component of the Travel Module that allows administrators to configure and manage travel services for both Business-to-Business (B2B) and Direct-to-Consumer (D2C) models. This system enables flexible pricing, commission structures, and service configurations for airlines and bus operators.

Key Features:
  • Dual configuration paths for B2B and D2C models
  • Centralized operator management through Travel Master
  • Flexible slab-based pricing with service charges and commissions
  • Smart operator selection with search and "All Operators" option

Configuration Steps

1 Access Master Module

Navigate to the Master Module in the admin panel and select "Product Service Master" to begin configuration.

2 Select User Type

Choose between B2B (Business-to-Business) or D2C (Direct-to-Consumer) based on the target audience for this service configuration.

3 Configure Basic Details

Select Product (Air API/Bus API), Category Type (Domestic/International), and configure the Operator using the searchable dropdown.

4 Set Pricing Structure

Define the Slab (price range with From Amount and To Amount) and configure Service Charge (Percentage/Amount/Not Applicable).

5 Configure Business Model Specifics

For B2B: Set Retailer Commission and specify retailer scope. For D2C: These fields are hidden as they don't apply.

6 Finalize and Save

Add the Inserted/Created Slab information and save the configuration to the Product Service Master Database.

B2B vs D2C Comparison

B2B Configuration
  • Includes retailer selection field
  • Requires retailer commission configuration
  • Targeted at business partners and resellers
  • Allows different pricing for different retailers
  • More complex configuration with partnership terms
D2C Configuration
  • No retailer selection (field hidden)
  • No retailer commission (field hidden)
  • Direct to end consumer pricing
  • Simplified configuration process
  • Focus on customer-facing service charges only

Operator Selection Logic

The Operator Selection uses a smart searchable dropdown with the following logic:

  1. Search Functionality: Administrators can type to find specific operators
  2. Specific Operator Selection: When an operator is found and selected, the configuration applies only to that specific operator
  3. "All Operators" Option: If the desired operator isn't found or for broad application, select "All Operators" to apply the configuration to all current and future operators matching the Product and Category Type
  4. Dynamic Filtering: Operators are filtered based on the selected Product (Air/Bus) and Category Type (Domestic/International)

Operator Management

New airlines and bus operators are added through the Travel Master section within the Master Module. This centralized approach ensures consistency across all product service configurations.
